# Specification index

Specifications are acceptance contracts for a feature or remediation at the
time it was designed. They are not automatically current operating guidance.
Use the [documentation maintenance guide](../development/documentation-maintenance.md)
and current guides first when behaviour has changed. A superseded specification
must retain its history and link to its successor.

Current cross-cutting contracts include [documentation accuracy](documentation-and-deployment-accuracy.md),
[customer search and routing](customer-search-routing-orchestration.md),
[bounded advisory planners](bounded-advisory-planners.md),
[LiteLLM connectivity](litellm-proxy-connectivity.md), and the
[22 July security remediation](security-scan-remediation-2026-07-22.md).
